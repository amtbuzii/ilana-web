# einat-web

AH-64D Apache helicopter mission planner — web reimplementation of the Einat VB6 desktop tool.

**React frontend** (port 5173) + **FastAPI backend** (port 8000) + **offline map/elevation data**.

---

## Quick Start

```bash
cd einat-web
./start.sh
```

Open **http://localhost:5173** in a browser.

The script creates a Python venv, installs all dependencies, and starts both servers. Press Ctrl+C to stop both.

Data lives outside the repo at `../data/` (relative to the `einat-web/` directory):

```
../data/
├── tiles/              Carto Voyager PNG tiles  (zoom 0–11)
├── topo_tiles/         OpenTopoMap PNG tiles    (zoom 0–14)

├── dem_tile_cache/     Pre-generated elevation heatmap PNGs (auto-populated)
└── srtm.vrt            GDAL VRT mosaic over dem_tiles/
```

## Frontend

### Key state (App.jsx)

| State | Description |
|-------|-------------|
| `routes[]` | Array of route objects, each with its own waypoints, stores config, and weights |
| `activeRouteId` | Which route is currently being edited/viewed |
| waypoints | Array of `{lat, lon, alt, airspeed, oat, atf, ...}` per route |
| `results` | Response from `/api/calculate` for the active route |
| `bgRoutes` | All non-active routes passed to MapView for background display |

### Map modes (MapView.jsx)

| Button | Tile source | Max zoom |
|--------|-------------|----------|
| MAP  | Carto Voyager (served locally) | 11 |
| TOPO | OpenTopoMap (served locally) | 14 |
| ELEV | DEM heatmap generated by backend | 11 |

### Aircraft variants

- **LB** — AH-64D Longbow (default)
- **Peten** — AH-64A
- **ALPHA** — AH-64A with ALPHA correction tables

## Backend

### API endpoints

**Flight planning**

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/calculate` | Full flight plan — returns per-leg fuel burn, torque, PA, OGE/IGE feasibility |
| POST | `/api/csp/fuel-from-oge` | Fuel solver: find fuel load that meets OGE torque target |
| POST | `/api/csp/fuel-from-ige` | Fuel solver: find fuel load that meets IGE torque target |

**Elevation**

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/elevation` | Ground elevation in feet at `?lat=X&lon=Y` |
| POST | `/api/terrain-profile` | Elevation samples along a multi-leg route |
| GET | `/api/dem-tiles/{z}/{x}/{y}.png` | Hypsometric heatmap tile (256×256 PNG) |

**Wing stores**

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/drag/config` | Available stores, presets, station labels |
| POST | `/api/drag/compute` | ATF drag multiplier from 4-station configuration |

**Coordinate conversion**

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/utm-to-latlon` | UTM (easting/northing) → WGS84 |
| POST | `/api/latlon-to-utm` | WGS84 → UTM (auto zone detection) |

**Status**

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/data-status` | Offline data availability summary |
| GET | `/api/data-coverage` | Tile coverage breakdown by zoom level |
| GET | `/health` | Health check |

**Static files**

The backend mounts `../data/` at `/data/`, so map tiles are served at:
- `http://localhost:8000/data/tiles/{z}/{x}/{y}.png`
- `http://localhost:8000/data/topo_tiles/{z}/{x}/{y}.png`

### Performance model (parsers.py, performance.py)

Binary table files (`oge`, `pa`, `cruise`, `cruiseLB`, `paLB`, `paALPHA`, `ffALPHA`, `atf`) are read from `../data/` at startup. These are the same lookup tables used by Einat.exe, ported from VB6.

- `OGETable` — Out-of-Ground-Effect hover torque required
- `PATorqueTable` / `LBTorqueTable` — Power Available by altitude and temperature
- `CruiseTable` — Cruise fuel flow and torque
- `OGETable._ogfnc()` — Bilinear interpolation (matches VB6 `OgFnc`)
- `rcfnc()` — Rate-of-climb torque correction (matches VB6 `RcFnc`)

### DEM heatmap tiles (main.py)

Elevation heatmap tiles are generated on demand with rasterio + NumPy + Pillow:

1. `_tile_bounds(z, x, y)` converts tile indices to a lon/lat bounding box
2. `_render_dem_tile()` reads a 256×256 window from the SRTM VRT, applies the hypsometric color ramp, and returns a PNG
3. Tiles are cached to `../data/dem_tile_cache/` on first request
4. Zoom levels 0–8 are pre-warmed in a background thread at startup

## Offline data setup

All map tiles and elevation data must be downloaded once before first use. The scripts below store data to `../data/` (one level above `einat-web/`).

### Map tiles (download_tiles.py)

```bash
# Road map — UTM zones 34–40 (lon 17–60°E), zoom 0–11 (~800 MB)
python3 download_tiles.py

# Topographic map — same region, zoom 0–12 (~2 GB)
python3 download_tiles.py --style topo

# Topographic map at higher detail — zoom 0–14 (~25 GB, takes hours)
python3 download_tiles.py --style topo --zoom 14
```

Tiles are served by the FastAPI backend via the `/data/tiles/` and `/data/topo_tiles/` static mounts.

### Elevation data (download_dem.py)

```bash
# SRTM3 90 m — UTM zones 34–40, ~1952 land tiles, ~5.4 GB
python3 download_dem.py

# Dry run: count tiles only
python3 download_dem.py --check

# Rebuild VRT from tiles already on disk (after manual download or move)
python3 download_dem.py --vrt-only
```

After download, restart the backend so it picks up the new `srtm.vrt`.

## Development

### Backend only

```bash
cd backend
source venv/bin/activate
uvicorn app.main:app --reload --port 8000
```

### Frontend only

```bash
cd frontend
npm install
npm run dev
```

### Tests

```bash
cd backend
source venv/bin/activate
pytest
```

### Production build

```bash
cd frontend
npm run build      # output → frontend/dist/
```

The `dist/` directory can be served as static files from the FastAPI backend or any static host.

## Dependencies

**Backend** (`backend/requirements.txt`):
- `fastapi` — Web framework
- `uvicorn` — ASGI server
- `numpy` — Numerical arrays for performance tables and DEM rasterization
- `pyproj` — UTM ↔ WGS84 coordinate conversion
- `pydantic` — Request/response validation
- `rasterio` — Reads SRTM HGT / VRT files
- `Pillow` — Generates elevation heatmap PNG tiles

**Frontend** (`frontend/package.json`):
- `react` / `react-dom` — UI framework
- `leaflet` + `react-leaflet` — Interactive map
- `protomaps-leaflet` — Vector tile support
- `xlsx` — Excel import/export
- `vite` — Build tool / dev server
